    • -DCLK_TCK
    • (?:^|(?<=\s))[-/]DCLK_TCK(?:=\S*)?(?=\s|$)

    • System clock ticks per second

    • -DDEBUGGING
    • (?:^|(?<=\s))[-/]DDEBUGGING(?:=\S*)?(?=\s|$)

    • Turn on output for debugging

    • -DDEFAULT_BND_VIOL
    • (?:^|(?<=\s))[-/]DDEFAULT_BND_VIOL(?:=\S*)?(?=\s|$)

    • Sets the tolerance for bounds violations based on precision level

    • -DDEFAULT_EPS_ZERO
    • (?:^|(?<=\s))[-/]DDEFAULT_EPS_ZERO(?:=\S*)?(?=\s|$)

    • Defines additive zero. 1.0 + EPS_ZERO == 1.0

    • -DFULL_SHIFT
    • (?:^|(?<=\s))[-/]DFULL_SHIFT(?:=\S*)?(?=\s|$)

    • Defines a different method for perturbation analysis

    • -DTEST
    • (?:^|(?<=\s))[-/]DTEST(?:=\S*)?(?=\s|$)

    • Enable a test to aid in debugging

    • -DTRACE_METHOD
    • (?:^|(?<=\s))[-/]DTRACE_METHOD(?:=\S*)?(?=\s|$)

    • Yet another debugging define

    • -DTWO_LOOPS
    • (?:^|(?<=\s))[-/]DTWO_LOOPS(?:=\S*)?(?=\s|$)

    • Enables a piece of code with two loops that is much slower than the default path.

    • -DUSE_OLD
    • (?:^|(?<=\s))[-/]DUSE_OLD(?:=\S*)?(?=\s|$)

    • USE_OLD allows the use of old, buggy, optimization-dependent code

    • -DWITH_LONG_DOUBLE
    • (?:^|(?<=\s))[-/]DWITH_LONG_DOUBLE(?:=\S*)?(?=\s|$)

    • WITH_LONG_DOUBLE declares real variables as long double rather than double

    • -DWITH_L_ROWS
    • (?:^|(?<=\s))[-/]DWITH_L_ROWS(?:=\S*)?(?=\s|$)

    • WITH_L_ROWS is always defined as 1
